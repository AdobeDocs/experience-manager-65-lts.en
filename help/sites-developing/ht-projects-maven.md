# How to build AEM Projects using Apache Maven {#how-to-build-aem-projects-using-apache-maven}

AEM 6.5 follows the latest best practices for package management and project structure. It uses the latest AEM Project Archetype for both on-premises and AMS implementations.

>[!TIP]
>
>For further details, see the following:
>
>* The [AEM Project Structure](https://experienceleague.adobe.com/en/docs/experience-manager-cloud-service/content/implementing/developing/aem-project-content-package-structure) article in the AEM as a Cloud Service documentation for how to structure modern AEM Projects.
>* The [AEM Project Archetype](https://experienceleague.adobe.com/en/docs/experience-manager-core-components/using/developing/archetype/overview) documentation for how to start a new AEM project using the archetype.
>* The [Adobe Content Package Maven Plugin](https://experienceleague.adobe.com/en/docs/experience-manager-cloud-service/content/implementing/developer-tools/maven-plugin#developer-tools) article in the AEM as a Cloud Service documentation for how to deploy AEM applications.
>
>All three documents apply to AEM 6.5.
